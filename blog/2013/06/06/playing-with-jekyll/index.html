"Playing With Jekyll"

Thu 06 June 2013

If you decide to start blogging with Jekyll-Bootstrap, your initial workflow might be write, push to github, and load the page in your web browser of choice. As I was learning how it worked, this became quite tedious (learning by pushing might not be the most efficient method....). As I did, you might find it easier to preview your posts with a local instance of Jekyll, rather than writing/pushing/reloading. Turns out, it is quite easy.

Getting Started with Jekyll

There are several good sites on getting started with blogging on github, but here is a good post (go down the rabbit hole of org-mode at your risk). To summarize how to make your first blogpost:

  • Clone the jekyll bootstrap repo into a directory of you choice. I'm using Dropbox, so I can edit posts on any of my machines or iOS devices
  • Update the repo to use your github account and update the config file ( _config.yml ) with your personal information
  • Create a new blog post:
    rake post title="your title"
    • Fire up vim or your favorite markdown editor and write the world's greatest blogpost
    • Push the repo to github

Voila! You're published!

Testing Makes Perfect

So, as I stated earlier, you can write/commit/push/reload until your heart's content. As you edit, play with themes, or otherwise tinker with your site, you'll find this process tiresome. Installing your own version of Jekyll will save you lots of time. If you're running on OSX, installing a local copy of jekyll is very easy.

  • Install Homebrew
  • Install Ruby from Homebrew
  • Use gem from Ruby to install Jekyll

ruby -e "$(curl -fsSL https://raw.github.com/mxcl/homebrew/go)"
brew install ruby
gem install jekyll 1

Now that jekyll is installed, you'll have to find the location ( I used the default location of /usr/local for homebrew, so my gem was installed under /usr/local/Cellar/... ). I couldn't get use the brew-gem installation of jekyll to work properly, which would symlink the binaries to somewhere in your path. Your path may be slightly different, but you'll start Jekyll in server mode in the directory of your cloned jekyll repo (from step 1 above):

[msbritt@ghost]$ cd ~/Dropbox/repos/msbritt.github.io
[msbritt@ghost]$ /usr/local/Cellar/ruby/2.0.0-p195/bin/jekyll serve

Configuration file: /Users/msbritt/Dropbox/repos/msbritt.github.io/_config.yml
Deprecation: Auto-regeneration can no longer be set from your configuration file(s). Use the --watch/-w command-line option instead.
Source: /Users/msbritt/Dropbox/repos/msbritt.github.io
Destination: /Users/msbritt/Dropbox/repos/msbritt.github.io/_site
Generating... done.
[2013-06-07 11:13:58] INFO WEBrick 1.3.1
[2013-06-07 11:13:58] INFO ruby 2.0.0 (2013-05-14) [x86_64-darwin12.3.0]
[2013-06-07 11:13:58] INFO WEBrick::HTTPServer#start: pid=57799 port=4000

Point your browser to localhost:4000 and voila, a local replica of your site. You can now experiement with settings, themes, and of course content, before pushing your content up for public consumption. Pretty cool!

  1. Using the Homebrew version of gem doesn't require any extended priviledges via sudo or root. ↩
